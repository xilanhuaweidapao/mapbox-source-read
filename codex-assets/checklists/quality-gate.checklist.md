# Quality Gate Checklist

所有项通过后才可标记完成。

## 1. 需求与范围

- [ ] 目标明确且单一
- [ ] In scope / Out of scope 明确
- [ ] 文件修改边界明确

## 2. 实现质量

- [ ] 代码遵循既有项目风格
- [ ] 关键接口有契约定义
- [ ] 空态/错态/重试齐全
- [ ] 权限受限行为符合预期

## 3. 视觉与体验

- [ ] 分辨率适配通过（大屏或网页对应标准）
- [ ] 交互反馈完整
- [ ] 信息层级清晰

## 4. 可访问性

- [ ] 键盘可达
- [ ] 焦点可见
- [ ] 语义标签合理
- [ ] 对比度可读
- [ ] 表单错误提示清晰

## 5. 性能与稳定性

- [ ] 首屏与关键交互满足预算
- [ ] 无明显内存异常增长
- [ ] 弱网/失败/超时可恢复

## 6. 测试与证据

- [ ] lint 通过
- [ ] 单元测试通过
- [ ] 集成测试通过
- [ ] 构建通过
- [ ] 输出验证命令与关键结果证据
